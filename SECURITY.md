# 安全性

GitHub 非常重視我們軟體產品與服務的安全性，包括透過我們的 GitHub 組織 (例如 [GitHub](https://github.com/GitHub)) 管理的所有開放原始碼儲存庫。

儘管[開放原始碼儲存庫不在我們的漏洞獎勵計畫 (bug bounty program) 範圍內](https://bounty.github.com/index.html#scope)，因此不具備獲得獎金的資格，但我們會確保將你的發現轉交給適當的維護者進行修復。

## 回報安全性問題

如果你認為在任何 GitHub 擁有的儲存庫中發現了安全漏洞，請透過協調披露 (coordinated disclosure) 的方式向我們回報。

**請勿透過公開的 GitHub issue、討論或提取請求 (PR) 回報安全漏洞。**

相反地，請發送電子郵件至 opensource-security[@]github.com。

請在郵件中包含儘可能詳細的資訊，以幫助我們更好地理解並解決問題：

  * 問題類型 (例如：緩衝區溢位、SQL 插入或跨網站指令碼)
  * 與問題表現相關的來源檔案完整路徑
  * 受影響原始碼的位置 (標籤/分支/提交或直接 URL)
  * 重現問題所需的任何特殊設定
  * 重現問題的逐步說明
  * 概念驗證 (PoC) 或利用程式碼 (如果有的話)
  * 問題的影響，包括攻擊者可能如何利用該漏洞

這些資訊將幫助我們更快速地對你的報告進行分類。
